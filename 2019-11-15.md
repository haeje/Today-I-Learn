# 클린코드 5장 형식 맞추기

5장은 결국 가독성의 이야기인 것 같다. 같이 협업하는 팀원, 추후에 변경사항으로 내가 짠 코드를 열어볼 개발자가 더 쉽게 코드를 이해할 수 있도록 코드를 짜야한다는 내용이다. 

이번 장에서 새로 알게 된 내용은 

## 코드의 가로 길이는 일반적인 모니터에서 한 줄에 다 표현될 정도가 이상적이라고 한다.
상식적이다. 가로 스크롤을 사용하며 코드를 봤던 경험은 최악이었다. 요즘에는 큰 모니터를 사용하고 글자도 축소해서 보는 경향이 있지만, 120자 정도가 적당하다고 한다.

## 또한 종속성이 있는 함수가 나열 될 때는 '호출하는 함수'가 '호출되는 함수'보다 앞에 작성되어 있는게 좋다.
알고리즘 문제를 풀 때 이상하게 찝찝했던 부분이다. 함수 A, B를 작성하고 A함수 내부에서 B함수를 호출한다고 하자. 이 때, 코드 상에서 A와 B 위치는 그때그때 달랐다. 

```java
B(){
  ...
}

A(){
  B();
 } 
 ```
 
 ```java
 A(){
  B();
 }
  
 B(){
  ...
}
```

글을 위에서 아래로 읽으며 편한 것은 후자이다. 위에서 아래로 글을 읽어내려가며 고수준에서 저수준으로 자연스럽게 읽힌다. 그리고 A 코드를 먼저 읽으면, 사람은 자연스럽게 뒤에 B코드가 기술되어 있을 것을 예상한다.

따라서, 호출하는 코드를 먼저 작성하면 된다.
